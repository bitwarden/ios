# Testing Checklist — Bitwarden iOS

Bitwarden-specific test requirements. See `Docs/Testing.md` for full guidelines.

## Test Infrastructure

- [ ] Tests use `ServiceContainer.withMocks(...)` — not manual mock construction
- [ ] Mock objects use `Mock*` prefix and are generated by Sourcery
- [ ] New protocols include `// sourcery: AutoMockable` annotation
- [ ] No real services or network calls in tests

## Test Coverage

- [ ] New `Processor` methods have corresponding unit tests
- [ ] Both `receive(_:)` (actions) and `perform(_:)` (effects) paths tested
- [ ] Error paths tested (not just happy path)
- [ ] State mutations verified with `XCTAssertEqual` on `subject.state`

## Snapshot Tests

- [ ] New/modified views have snapshot tests in three modes: `.defaultPortrait`, `.defaultPortraitDark`, `.defaultPortraitAX5`
- [ ] Snapshot test function names prefixed with `disabletest_` (snapshot tests currently disabled globally)
- [ ] Simulator matches `.test-simulator-device-name` and `.test-simulator-ios-version`

## Test File Co-location

- [ ] Test files live alongside implementation files (not in a separate `Tests/` directory tree)
- [ ] Test file named `<Type>Tests.swift` or `<Type>+<Category>Tests.swift` matching the production file

## Naming Convention

- [ ] Test methods follow `test_<functionName>_<behaviorDescription>` pattern
- [ ] Test method names describe observable behavior, not implementation details
